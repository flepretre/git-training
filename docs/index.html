Git

Chapitre 1

Les espaces git

  • local
  • working directory
  • l'index

local

  • Git est un systeme de versionning distribué
  • chaque clone du repository est autonome
  • la majeure partie des commande git ne font pas de réseau

local

  • pour créer un repo git en local
git init
  • crée un répertoir .git qui va contenir toute la config git

working directory

  • la racine de votre repo est le working directory
  • il représente l'état courant du code
  • lors qu'il est modifié, git détecte les différences
git diff

l'index

  • zone dans laquelle git référence les changements à commiter
git status
  • pour ajouter des changements dans l'index
git add [<options>...] [<pathspec>...]

Les opérations de branch

  • cherry-pick
  • merge
  • rebase

Cherry-pick

  • permet de récupérer des commits sur la branche courante
  • les commits sont dupliqués
  • la branche courante progresse
  • risque de conflits
  • en cas de conflits, la résolution se trouve dans le commit récupéré
  • si le commit est vide (pas de diff car le code est déjà présent), il est ignoré

Cherry-pick

git cherry-pick toto ad123edf

Merge

  • permet de fusioner des branches
  • notion de fast-forward
  • si fast-forward, déplace simplement la branche courante
  • si non fast-forward, crée un commit avec plusieurs parents (commit de merge)
  • risque de conflits
  • en cas de conflits, la résolution se trouve dans le commit de merge

Merge

git merge foo
git merge foo --ff-only
git merge foo --no-ff

Rebase

  • déplace une branche
  • tous les commits sont dupliqués à leur nouvelle position (car immuables)
  • risque de conflits
  • en cas de conflits, la résolution se trouve dans le commit récupéré
  • si le commit est vide (pas de diff car le code est déjà présent), il est ignoré

Rebase

git rebase foo
git rebase foo -i

Chapitre 5

Le réseau

  • remote
  • fetch
  • push
  • pull

Le réseau

Tout ce qu'on a vu jusqu'à présent est 100% offline

remote

  • permet de définir les urls pour des repos distant
  • il est possible d'avoir deux repos differents pour le fetch et le push
  • il est possible d'avoir plusieurs repos distant

remote

git remote -v
git remote add flepretre https://github.com/flepretre/m6git
git remote set origin https://github.m6web.fr/m6web/m6git

fetch

  • récupère l'état des repos distants
  • fait progresser les branches distantes (origin/toto)
  • permet aussi de récupérer les tags
  • n'a aucun impact sur les branches locales

fetch

git fetch
git fetch flepretre
git fetch --tags
git fetch --prune

push

  • pousse des commits sur un repo distant
  • fait progresser la branche distante
  • échoue si la progression de la branche n'est pas fast-forward

push

git push origin master
git push origin master~:master -f
git push origin --tags
git push origin :toto -f

pull

  • fait un fetch suivit d'un merge
  • ⚠️ par défaut, si le merge est non fast-forward, un commit de merge est créé
