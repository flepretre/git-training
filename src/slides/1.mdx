# Les espaces git

* local
* working directory
* l'index
* la config

---

## local

* Git est un systeme de versionning distribué
* chaque clone du repository est autonome
* la majeure partie des commandes git ne font pas de réseau

```notes
- qu'est-ce que le versioning
- pourquoi versioner
- versioning distribué
```

---

## local

* pour créer un repo git en local

```sh
git init
```

* crée un répertoir `.git` qui va contenir toute la config git

---

## working directory

* la racine de votre repo est le working directory
* il représente l'état courant du code
* lors qu'il est modifié, git détecte les différences

```sh
git diff
```

---

## l'index

* zone dans laquelle git référence les changements à commiter

```sh
git status
```

* pour ajouter des changements dans l'index

```sh
git add [<options>...] [<pathspec>...]
```

```notes
- possibilité d'ajouter ligne par ligne
```

---

## La config

* il est possible de configurer certaines variables de git
```sh
git config user.email foo@bar.fr
git config user.nam "foo bar"
```

---

## La config

* la config de chaque repo est disponible dans le fichier ".git/config"
* il est aussi possible de configurer un fichier de config globale
